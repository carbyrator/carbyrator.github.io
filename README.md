# Отчет по структуре проекта и настройке деплоя

В рамках задания был подготовлен репозиторий со статическим сайтом-портфолио на MkDocs. Были созданы основные страницы сайта: главная, «Обо мне», обзор лабораторных работ и отдельные страницы с описанием проектов. Также была подключена пользовательская таблица стилей для дополнительного оформления.

Репозиторий организован следующим образом: в папке source/docs/ находятся все Markdown-страницы сайта, в файле source/mkdocs.yml описаны настройки сборки и навигация, а папка source/site/ содержит готовый результат сборки. В корне репозитория размещены служебные файлы для автоматизации деплоя: .github/workflows/ для GitHub Actions и .sourcecraft/ для Sourcecraft.

Для автоматизации настроены два варианта деплоя. Через GitHub Actions выполняется сборка сайта и публикация в GitHub Pages. Через Sourcecraft сайт также собирается автоматически, после чего результат переносится в папку site и публикуется из ветки release.

Чтобы выполнить деплой, необходимо загрузить актуальные изменения в основную ветку репозитория, после чего автоматически запускается сборка. Если workflow завершается без ошибок, сайт публикуется либо через GitHub Pages, либо через Sourcecraft Sites.

В настройках GitHub репозитория необходимо было включить GitHub Pages, выбрать источник публикации GitHub Actions и убедиться, что для workflow доступны права на публикацию страниц. В настройках Sourcecraft необходимо было подключить репозиторий, включить выполнение workflow и указать публикацию сайта из ветки release и папки site.

В результате был получен репозиторий с понятной структурой, автоматической сборкой и настроенным деплоем сайта.
