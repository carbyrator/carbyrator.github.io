# Отчёт по лабораторной работе

## Цель

Целью лабораторной работы было построить модель классификации для предсказания кредитного дефолта клиента по финансовым признакам.  

В качестве целевой переменной использовался признак **SeriousDlqin2yrs**, который показывает, был ли у клиента серьёзный дефолт.

---

## Задание

В ходе работы были загружены и подготовлены обучающая и тестовая выборки.  

Из таблиц были выделены признаки и целевая переменная, после чего были обучены две модели классификации:
- Logistic Regression  
- Random Forest  

Далее для обеих моделей были получены предсказания на тестовой выборке и рассчитаны основные метрики качества.

---

## Анализ качества моделей

Сначала качество моделей оценивалось по **accuracy**, однако эта метрика оказалась недостаточно показательной, так как в данных наблюдается дисбаланс классов: случаев без дефолта значительно больше, чем случаев с дефолтом.  

Поэтому дополнительно были использованы:
- confusion matrix  
- анализ порога классификации  
- метрика ROC-AUC  

Анализ показал, что:
- Logistic Regression чаще склонна относить объекты к классу 0 и хуже выявляет дефолты  
- Random Forest показывает более качественное разделение классов  

Также было рассмотрено влияние порога классификации на количество ошибок:
- False Positive  
- False Negative  

Это особенно важно для задач кредитного скоринга.

---

## ROC-анализ

Отдельно была построена ROC-кривая и рассчитана метрика ROC-AUC, которая позволила более объективно оценить качество моделей.  

По результатам основной части работы модель **Random Forest** показала более высокий результат, чем Logistic Regression, и оказалась более подходящей для данной задачи.

---

## Самостоятельная работа

В самостоятельной части были исследованы другие модели классификации:
- kNN  
- SVM  

### Результаты (ROC-AUC)

- Logistic Regression: около 0.69  
- Random Forest: около 0.84  
- kNN: около 0.73  

Модель kNN показала результат лучше логистической регрессии, но уступила Random Forest.  

Модель SVM на таком объёме данных обучается значительно дольше, поэтому в рамках данной работы её использование оказалось менее практичным.

Таким образом, улучшить результат относительно Random Forest не удалось. Это можно объяснить:
- дисбалансом классов  
- ограниченной информативностью признаков  
- тем, что для табличных данных модели на основе деревьев решений работают лучше

## Современные методы классификации

По современным исследованиям и практическим решениям в задачах кредитного скоринга часто используются:

- Random Forest  
- Gradient Boosting  
- XGBoost  
- LightGBM  
- CatBoost  

Это связано с тем, что такие методы:
- хорошо работают с нелинейными зависимостями  
- учитывают взаимодействие признаков  
- показывают высокое качество на табличных данных

## Интеграция модели в веб-сервис

Обученную модель можно интегрировать в веб-сервис на Flask, Django или FastAPI.  

Для этого необходимо:

1. Обучить и сохранить модель вместе с preprocessing pipeline  
2. Создать API-приложение  
3. Описать входные данные для запроса  
4. Загружать модель при запуске сервиса  
5. Реализовать endpoint для получения предсказания  
6. Добавить проверку данных и обработку ошибок  
7. Развернуть сервис на production-сервере

## Вывод

В ходе лабораторной работы были изучены основные этапы решения задачи бинарной классификации:
- подготовка данных  
- обучение моделей  
- получение предсказаний  
- анализ качества  

По итогам основной части работы лучшей моделью оказался **Random Forest**.  

В самостоятельной части были исследованы дополнительные модели. Было установлено, что:
- kNN не превосходит Random Forest  
- SVM слишком затратна по времени обучения  

Следовательно, для данной задачи наиболее подходящими являются модели на основе деревьев решений и их ансамблей.
